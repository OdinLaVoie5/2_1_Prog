2.7. CSS таблицы

Спецификация CSS дает неограниченные возможности для оформления таблиц.

По умолчанию таблица и ячейки таблицы не имеют видимых границ и фона, при этом ячейки внутри таблицы не прилегают вплотную друг к другу.

Ширина ячеек таблицы определяется шириной их содержимого, поэтому ширина столбцов таблицы может быть разной.

Высота всех ячеек сетки строки по умолчанию одинаковая.

1. Границы таблицы

Изначально таблица и ячейки внутри нее не имеют видимых границ. Границы таблицы оформляются свойством border:

CSS
1
2
3
4
table {
border-collapse: collapse; /*убираем пустые промежутки между ячейками*/
border: 1px solid grey; /*устанавливаем для таблицы внешнюю границу серого цвета толщиной 1px*/
}

Границы ячеек заголовка задаются для элемента th:

CSS
1
2
3
th {
border: 1px solid grey;
}

Границы ячеек тела таблицы задаются для элемента td:

CSS
1
2
3
td {
border: 1px solid grey;
}

Так как толщина рамок соседних ячеек при задании их способом border: 1px solid grey; не удваивается, поэтому, чтобы задать границы для всей таблицы, достаточно будет указать их для th, td {border: 1px solid grey;}.

Чтобы выделить внешнюю границу таблицы, нужно установить, например, table {border: 3px solid grey;}.

Также можно частично задавать границы как таблицы, так и ячеек, например:

CSS
1
2
3
4
5
6
table {
border-top: 3px solid grey; /*устанавливаем для таблицы внешнюю границу серого цвета толщиной 3px*/
}
td {
border-bottom: 1px solid grey;
}

Подробнее о свойстве border вы можете прочитать здесь.

2. Ширина и высота таблицы

По умолчанию ширина и высота таблицы определяется содержимым ее ячеек. Если ширина не задана, то она будет равна ширине самого широкого ряда (строки).

Ширину таблицы и ее столбцов можно задать с помощью свойства width. Если для таблицы задана ширина table {width: 100%;}, то она будет равна ширине блока-контейнера, а ширина столбцов установится в соответствии с шириной содержимого ячеек.

Чаще всего ширину таблицы и столбцов задают в px или %, например:

CSS
1
2
3
table {width: 600px;}
th {width: 20%;}
td:first-child {width: 30%;}

Чтобы отделить содержимое ячеек от рамок, нужно добавить для элементов <td> и <th> соответствующие поля — padding.

Высота таблицы обычно не задается. Высотой строк таблицы можно управлять, добавив верхний и нижний padding, или же задав высоту с помощью свойства height, например:

CSS
1
2
th, td {padding: 10px 15px;}
th, td {height: 30px;}

3. Фон таблицы

По умолчанию фон таблицы и ее ячеек прозрачный, и, если страница или блок, содержащие таблицу, имеют фон, то он будет просвечиваться сквозь таблицу.

В качестве фона для таблицы в целом и ее ячеек могут выступать:
заливка сплошным цветом,
градиент,
фоновое изображение.

При этом, если фон задан и для таблицы и для ячеек, то в местах наложения фона таблицы и ячеек будет виден фон только ячеек.

4. Столбцы таблицы

Не смотря на то, что модель таблиц CSS ориентирована в основном на строки, тем не менее столбцы играют важную роль в макете. Форматирование столбцов таблицы возможно следующими способами:

с помощью тега <col> можно задать фон для любого количества столбцов,

с помощью конструкции table td:first-child, table td:last-child можно задать стили для первого или последнего столбца таблицы (за исключением первой ячейки заголовка таблицы),

с помощью конструкции table td:nth-child(правило отбора столбцов) можно задать стили для любых столбцов таблицы.

5. Основной заголовок таблицы

Основной заголовок <caption> описывает суть содержимого таблицы. С помощью свойства caption-side его можно поместить перед таблицей или под ней. Для горизонтального выравнивания текста заголовка применяется свойство text-align. Наследуется.

caption-side
Значения:
top Заголовок таблицы располагается над таблицей. Значение по умолчанию.
bottom Располагает заголовок под таблицей.

6. Рамки ячеек таблицы

По умолчанию действует модель отдельных рамок, которую можно поменять на модель сливающихся рамок. Наследуется.

border-collapse
Значения:
collapse Рамки ячеек будут сливаться.
separate Рамки ячеек будут располагаться раздельно.

CSS
1
2
3
4
5
6
table {
border-collapse: collapse;
}
table {
border-collapse: separate;
}

7. Промежуток между рамками

Свойство позволяет задать расстояние между рамками ячеек таблицы. Свойство применяется к таблице в целом. Промежуток между рамками ячеек постоянен независимо от ширины рамок самих ячеек. Наследуется.

border-spacing
Значения:
<длина> <длина> Задание длины позволяет разделить ячейки промежутком как по вертикали, так и по горизонтали. Если заданы две длины, то первая всегда определяет горизонтальный промежуток, а вторая — вертикальный.

CSS
1
2
3
4
5
6
table {
border-spacing: 10px 20px;
}
table {
border-spacing: 10px;
}

8. Обработка пустых ячеек таблицы

Свойство дает возможность скрывать или показывать пустые ячейки. Действует только на ячейки, которые не содержат какой-либо контент, но если для ячейки задан фон, а для таблицы задано table {border-collapse: collapse;}, то она не будет скрыта. Наследуется.

empty-cells
Значения:
show Рамка и фон пустой ячейки будут отрисовываться так же, как для ячейки таблицы, имеющей содержимое.
hide Если все ячейки строки пусты, то вся строка отображается так, если бы было задано значение display: none.

9. Компоновка ширины таблицы

В общем случае ширина таблицы может определяться в соответствии с одним из двух подходов: компоновка с фиксированной шириной и компоновка с автоматической шириной. Высота таблицы вычисляется автоматически. Наследуется.

table-layout
Значения:
auto Ширина таблицы определяется или значением свойства width самой таблицы, или суммой ширины всех столбцов, т.е. тем, что имеет большее значение. Значения ширины всех столбцов определяются первой строкой таблицы. Если содержимое не помещается, значение свойства overflow ячейки определит, будет ли содержимое отсечено, будет ли оно видимым или будет генерироваться полоса прокрутки.
В случае table-layout: auto таблица стремится занять как можно меньше места по ширине, а полная ширина ячейки складывается из ширины ячейки (width) + padding-left + padding-right + border-left width и border-right width. При этом установленные ширины ячеек или ширина таблицы не окажут влияние на её отображение, если они таковы, что контент ячеек может быть полностью или частично скрыт.
Если задана слишком большая ширина таблицы, например table {width: 100%;}, то ширина всех ячеек будет равной.
fixed Таблица не будет планироваться до тех пор, пока браузер пользователя не просмотрит все ее содержимое. Если для таблицы задана ширина, например table {width: 100%;}, то ширина ячеек будет разной, при этом ячейки, имеющие одинаковое содержимое по ширине, будут одинаковой ширины.
